1. SponsorAssets centralizado — sistema de identidad visual
Contexto
En Swift implementamos SponsorAssets.swift como fuente única de verdad para la identidad visual del sponsor activo. Esta tarjeta es el port a Kotlin.
Swift reference: Sources/VioDesignSystem/Components/SponsorAssets.swift
Lo que hace el sistema Swift (a replicar en Kotlin)
Lee sponsor.logoUrl, avatarUrl, primaryColor, secondaryColor, name, badgeText del backend
Calcula textOnPrimary por luminosidad WCAG (0.2126R + 0.7152G + 0.0722B) — no depende del backend
SponsorAvatarComposable — avatar circular 32dp, Coil AsyncImage + fallback iniciales
Usado en: TimelinePollCard, ContestCard, ProductCarousel
Tarea — Port a Kotlin
Crear SponsorAssets.kt en VioDesignSystem con las mismas propiedades.

data class SponsorAssets(
    val sponsorId: Int,
    val name: String,
    val logoUrl: String?,
    val avatarUrl: String?,
    val primaryColor: String,
    val secondaryColor: String?,
    val badgeText: String?,
    val textOnPrimary: Color  // calculado por luminosidad WCAG
)
