1. VProductDetailOverlay — ocultar CTA si hay método de pago activo
Contexto
En Swift, VProductDetailOverlay tiene un botón estándar "Legg til i handlekurv".
Cuando el backend tiene apple_pay activo en checkoutConfig, ese botón se oculta y aparece VPaymentSheet en su lugar. La lógica es dinámica: el overlay observa VioConfiguration y reacciona cuando llega la config del backend.
Código Swift relevante:


@observedobject private var vioConfig = VioConfiguration.shared
if !vioConfig.checkoutConfig.hasApplePay {
    // mostrar botón addToCart
}
VPaymentSheet(...) // siempre presente
Lo que se espera en Kotlin
En VioProductDetailOverlay.kt (o el equivalente Composable):
Observar checkoutConfig de VioConfiguration
Si hasApplePay == true → ocultar el botón CTA estándar
Mostrar VPaymentSheet en su lugar
Referencia Swift: Sources/VioUI/Components/VProductDetailOverlay.swift

3. shouldShowComponent — componentes de campaña siempre visibles
Contexto
Había un bug en Swift donde el carousel de productos y el banner de sponsor desaparecían aunque la campaña estuviera activa. El problema era que shouldShowComponent ocultaba componentes de nivel campaña cuando broadcastContext aún no estaba seteado.
Fix aplicado en Swift (CampaignManager.swift, commit e8febaf):
Componentes sin broadcastContext (carousel, banner) → siempre visibles si la campaña está activa
Solo ocultar si el componente tiene un broadcastContext.broadcastId que no coincide con el broadcast activo


func shouldShowComponent(_ component: CampaignComponent) -> Bool {
    guard isCampaignActive else { return false }
    guard let componentBroadcastId = component.broadcastContext?.broadcastId else {
        return true // campaign-level component, always show
    }
    return componentBroadcastId == currentBroadcastContext?.broadcastId
}
Lo que se espera en Kotlin
Revisar la lógica equivalente en Kotlin SDK (probablemente en CampaignManager.kt):
Confirmar que componentes sin broadcastContext se muestran siempre
Si la lógica actual oculta todo cuando broadcastContext es null → corregir
Referencia Swift: Sources/VioCore/Managers/CampaignManager.swift método shouldShowComponent
