1. Los componentes de producto (banner, carousel) deben aceptar un locationId para que el operador pueda asignarlos desde el dashboard sin tocar codigo.
Que hacer:
Anadir locationId: String? al modelo Component
Metodo en CampaignManager: getActiveComponent(locationId: String)
VProductBanner y VProductCarousel aceptan locationId como parametro
Si viene locationId, busca por slot. Si no, usa componentId como fallback.
Slots estandar: sport-detail-banner, sport-detail-carousel
Referencia Swift: Sources/VioUI/VProductBanner.swift

Que hacer:
ProductService lee dynamicCommerceConfig?.apiKey
Si no hay apiKey: lanzar error claro ('Commerce API key not configured')
VioConfiguration.updateDynamicCommerceConfig() al recibir la config
Notificacion de cambio para que los componentes se actualicen
